Design System · v0.1

Device Pulse

A visual language for reading the health of a live telemetry connection at a glance — built for a multiplatform sensor dashboard running on Android, iOS, and Desktop.

01Color

A graphite instrument panel rather than pure black — warm enough to hold long monitoring sessions. Two signal accents carry meaning, never decoration: amber for anything live or attention-worthy, cyan for stable and nominal.

02Type

A complete scale, not a sampler — every text role the app actually uses gets a name, and nothing gets sized ad hoc on a screen-by-screen basis. Space Grotesk carries titles and UI chrome; IBM Plex Sans carries prose; IBM Plex Mono is reserved strictly for anything that is measured data — readings, timestamps, topic strings, log entries — so numbers read as instrumentation, not styled copy.

04Payload contract

Each device publishes a connection status that drives the connection chip UI. One MQTT message = one sensor reading, plus the stream's current connection status.

Live
"status": "live"
Sensor is publishing actively. Cyan dot + nominal value.
Reconnecting
"status": "reconnecting"
Stream lost, broker retrying. Amber blinking dot, last known value dimmed.
Offline
"status": "offline"
No data. Dim dot, value shown as last known or "—".

07The signature element

The signal trace is the one motif repeated everywhere a connection appears — dashboard header, per-device row, detail view, desktop panel. It's not decorative: a flat, tight line means stable; a jagged one means the connection is alive but unhealthy; a dashed flat line means no data at all. A user should be able to tell those three apart without reading a single word, the same way you'd read an oscilloscope.
